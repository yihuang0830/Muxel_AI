# 2026-05-02 开发日志

## 今天做了什么

### 1. Step 2 歌单页面持续优化

- **备选歌单**：`generate()` 用 Last.fm 剩余候选（按热度排序，最多 20 首）自动填充备选列表，每首右侧有 ＋ 按钮加入正式歌单
- **Last.fm 状态可视化**：主题 caption 显示 `Last.fm ✅ N 首候选（tag1、tag2）` 或 `Last.fm 未启用`，方便确认数据库是否生效
- **表格与小张对话同步**：修复模糊匹配（去书名号/标点/空格），修复 `max_tokens` 不足导致 JSON 截断的问题，让排序/删除指令能正确更新左边表格

### 2. Step 3：主持人像素写稿

- `pipeline/step3_script.py`：像素 agent，读取 `01_topics.md`（选题方向）+ `02_playlist.json`（确认歌单），生成完整中文主持稿
- 输出格式：开场白 → 每首歌介绍 + `[MUSIC: 歌名 - 艺人]` 标注 + 过渡语 → 结语
- UI：生成后在可编辑文本框展示，支持直接修改；「重新生成」按钮让像素再写一版
- 目前只做中文版，英文版后续再加

### 3. 步骤间自由导航

- 进度条中已完成的步骤（✅）变为可点击按钮
- 点击任意已完成步骤可回跳，数据文件保留不重置
- 可在稿本页直接跳回歌单或选题进行修改
