# 2026-04-24 开发日志

## 今天做了什么

### 1. 确定多 Agent 架构
- 每个流水线步骤对应一个独立 AI 角色，共用一个 Qwen API
- 确定角色阵容：小柒（助理/Step 1）、小张（乐评人/Step 2）、像素（主持人/Step 3）

### 2. Step 2：乐评人小张
- `pipeline/step2_playlist.py`：小张 agent 完整实现
  - `generate()`：生成初始歌单
  - `chat_reply()`：对话中处理增删改排序，返回结构化 JSON
- 支持操作：加歌、删歌、重新排序（`reorder_titles`）
- 对话消息即时渲染（用户按下 Enter 立刻显示，不等 LLM）

### 3. Last.fm 音乐数据库集成
- `pipeline/music_db.py`：封装 Last.fm API
  - `extract_tags_from_direction()`：从选题方向提取风格 tag
  - `search_by_tags()`：按 tag 拉取真实候选曲目
  - `get_artist_top_tracks()`：获取艺人代表曲
  - `verify_track()`：验证歌曲存在并补全专辑信息
- 生成歌单流程：方向 → 提取 tag → Last.fm 候选池（50首）→ 小张策划选10首
- 对话加歌时自动验证曲目真实性，补全艺人/专辑名
- Last.fm 不可用时自动降级为纯 LLM 模式

### 4. Step 2 UI 重新设计
- 双栏布局：左边只读歌单表格，右边和小张的聊天框
- 表格由小张全权维护，用户只看，不直接编辑
- 聊天框高度固定，消息区可滚动
